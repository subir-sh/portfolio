Project Overview

AreUComing is a mobile web service for couples that shares routes in real time while they are on the way to meet. Instead of treating location sharing as passive tracking, the service was designed around the journey itself and the moment of meeting as a shared experience.

Developed during WaffleStudio's 2026 Wackathon, the project won 3rd place. I mainly worked on backend-side session modeling and real-time flow, focusing on how live location updates, session history, and authenticated access could work together reliably.

What I Did

  • Modeled the database schema for session-based location sharing, including sessions and session point history.
  • Implemented backend and frontend WebSocket flow for real-time session updates.
  • Connected JWT authentication to the WebSocket handshake and added session membership validation.
  • Combined HTTP-based history loading with live socket updates so the map could recover state after refresh and continue streaming afterward.
  • Handled automatic socket start on session creation and acceptance, along with session completion flow.

Design Challenge

The main challenge was designing real-time location sharing around an application-level session lifecycle rather than around the socket connection itself. In this service, a WebSocket was only one transport layer inside a larger flow that also included authenticated session entry, history loading, live point streaming, and session completion.

I worked on structuring that flow so that past session data could be loaded through HTTP, new updates could stream through WebSocket, and both would remain consistent across refreshes and reconnects. This made the backend closer to a session-oriented real-time system than a simple live socket demo.
