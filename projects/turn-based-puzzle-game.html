Project Overview

Chronos is a Unity-based 3D puzzle game designed around turn-based progression and a rewind mechanic that creates clones. The project was developed as a university team project with the goal of delivering a short but complete puzzle gameplay experience.

I focused on gameplay-side systems: turn flow, object interaction, and game state related refactoring. Rather than only adding features, I worked on keeping puzzle behavior consistent as interaction rules grew more complex.

What I Did

  • Implemented turn progression and player-object interaction systems.
  • Built gameplay-adjacent systems including save/load, scene transitions, and map flow.
  • Refactored state related gameplay logic to make turn execution and object responses easier to track and modify.
  • Worked in a sprint-based team workflow using GitHub, Wiki documentation, and Slack.
  • Received a 22% contribution score in anonymous peer evaluation within a five-member team.

Design Challenge

The main challenge in Chronos was resolving each turn in a consistent order across multiple interacting objects. A single input could affect not only the player, but also clone replay, box movement, lever-triggered changes, falling checks, and rewind logs.

Because these updates were order sensitive, small gameplay changes could easily introduce side effects in movement resolution or state restoration. I focused on making turn execution and state updates easier to follow and adjust without breaking object interactions.
