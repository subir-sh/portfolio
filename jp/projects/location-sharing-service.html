AreUComing

目的地そのものだけでなく、会うまでの移動過程も共有できる、セッション型のリアルタイム位置共有サービスです。

担当 · バックエンド開発、WebSocket通信フロー実装、セッション設計

使用技術 · Java, Spring Boot, React, WebSocket, Docker, MySQL, GitHub

関連リンク · バックエンド · フロントエンド · Figma

概要

AreUComing は、会うまでの移動ルートをリアルタイムで共有する、カップル向けのモバイルウェブサービスです。位置共有を単なる追跡として扱うのではなく、移動そのものと出会う瞬間を共有体験として捉え、サービスとして設計しました。

WaffleStudio の 2026 年 Wackathon で制作され、銅賞を受賞しました。私は主にバックエンド側のセッション設計とリアルタイム処理を担当し、位置更新、履歴管理、認証付きアクセスをどのように一貫して動かすかに取り組みました。

担当内容

  • セッション型位置共有のためのデータベーススキーマを設計し、sessions と session_points を中心としたデータモデルを構築しました。
  • リアルタイム更新のための WebSocket フローを、バックエンドとフロントエンドの両方で実装しました。
  • JWT 認証を WebSocket の接続処理に組み込み、セッション参加者の検証を追加しました。
  • HTTP による履歴取得とソケット更新を組み合わせ、再読み込み後も地図の状態を復元しながら継続利用できるようにしました。
  • セッション作成・受諾時の自動接続開始と、完了までの流れも実装しました。

設計上の課題

難しかったのは、リアルタイム位置共有を単なるソケット通信としてではなく、アプリケーション全体のセッションライフサイクルとして設計することでした。このサービスでは WebSocket は単なる通信手段の一つであり、その周囲に認証付き参加、履歴取得、位置ストリーミング、セッション完了といった流れが存在します。

私はその全体の流れを整理し、過去データは HTTP で取得し、新しい更新は WebSocket で配信しながら、再読み込みや再接続後も両者が自然につながるように構成しました。単なるリアルタイム通信の実験ではなく、セッション指向のリアルタイムシステムとして成立することを意識した実装です。
