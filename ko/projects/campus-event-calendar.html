프로젝트 개요

행샤는 서울대학교 비교과 행사 정보를 더 쉽게 찾고 관리할 수 있도록 만든 캘린더 서비스입니다. 기존 플랫폼에서 행사 정보를 찾기 어렵다는 문제를 보고, 캘린더 뷰, 필터링, 북마크, 시간표 연동, 메모 기능을 제공했습니다.

저는 백엔드 개발과 데이터 수집 파이프라인을 주로 맡았습니다. 외부 페이지에서 행사 정보를 수집하고 정규화해, 캘린더와 개인화 기능에서 바로 사용할 수 있는 형태로 제공했습니다.

담당 내용

  • 교내 행사 데이터를 수집하고 정규화해 API로 제공하는 흐름을 구현했습니다.
  • 캘린더 뷰, 검색, 행사 상세, 카테고리/그룹 조회 등 이벤트 관련 API를 구현했습니다.
  • 북마크, 제외 키워드, 메모, 태그 등 개인화 기능에 필요한 백엔드 로직을 구현했습니다.
  • 목록 페이지와 상세 페이지의 구조 차이에 맞춰 OkHttp와 Playwright를 함께 사용하는 크롤링 로직을 구성했습니다.

설계상 과제

행샤에서 어려웠던 점은 단순히 데이터를 긁어오는 것이 아니라, 구조가 일정하지 않은 외부 페이지를 안정적인 서비스 데이터로 바꾸는 일이었습니다. 기존 플랫폼에는 정적 페이지와 브라우저 실행이 필요한 페이지가 섞여 있었고, 접근 방식도 페이지마다 달랐습니다.

그래서 수집 로직과 서비스 API 계층을 분리하고, 페이지 특성에 따라 수집 방식을 나눴습니다. 수집한 결과는 하나의 이벤트 모델로 정규화해 프론트엔드에서 일관되게 사용할 수 있도록 했습니다.
