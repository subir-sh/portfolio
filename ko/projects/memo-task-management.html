프로젝트 개요

ThinkFlow는 메모, 태스크, 회고 포스트를 하나의 연결된 시스템에서 관리하기 위해 만든 풀스택 생산성 앱입니다. 떠오른 생각, 실행 과업, 회고 기록을 분리된 도구로 다루기보다 아이디어→실행→회고로 이어지는 연속 흐름을 유지하는 데 집중했습니다.

제품 구조 설계와 구현을 모두 맡아, 분절된 CRUD 페이지보다 공통 아이템 모델과 전환 흐름 중심으로 앱을 구성했습니다. 그 결과 메모/포스트를 태스크로 승격하고, 완료된 태스크를 회고 포스트로 이어가는 문서형 워크플로우를 구현했습니다.

담당 내용

  • 메모, 포스트, 태스크를 하나의 워크플로우로 관리하는 풀스택 웹 앱을 구현했습니다.
  • 핵심 CRUD 흐름과 검색·정렬·페이지네이션·태그 필터링 기능을 구현했습니다.
  • 타입 전환 시 사용자 맥락을 유지하도록 메모/포스트 → 태스크 승격 흐름을 설계·구현했습니다.
  • 모달 기반 초안 생성 경로를 포함한 태스크 → 회고 포스트 전환 흐름을 구현했습니다.
  • frontend/backend/shared 구조로 프로젝트를 분리하고, 구현 전 사용자 스토리와 기능 명세로 제품을 정의했습니다.

설계상 과제

핵심 과제는 메모·태스크·포스트 기능을 각각 만드는 것이 아니라, 하나의 일관된 아이템 생명주기로 연결하는 것이었습니다. 빠른 기록, 태스크 관리, 태깅, 회고 작성을 동시에 지원해야 했기 때문에 공통으로 둘 부분과 타입별로 분리할 부분을 정하는 일이 특히 중요했습니다.

공통 아이템 모델을 중심으로 설계하고 태스크 전용 속성을 레이어링했으며, 메모/포스트 → 태스크, 태스크 → 회고 전환 흐름에 맞춰 UI를 정렬했습니다. 이를 통해 기능을 서로 분리된 화면으로 취급하지 않고도 확장 가능한 구조를 만들었습니다.

미리보기

분리된 미니 앱이 아니라 하나의 연속된 워크플로우로 동작하도록 설계한 메모/포스트 화면과 태스크 관리 화면입니다.
